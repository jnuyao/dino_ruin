# Dino Math Ruins UI/UX Revamp Spec (V2)

## 1. 目标
在不改变当前核心流程（地图 -> 关卡 -> 回地图 / 孵蛋 -> 图鉴）的前提下，提升“第一眼吸引力”和“持续游玩欲望”，让页面更像“探险游戏”而不是“练习题页面”。

## 2. 设计原则
1. 视觉叙事统一：遗迹、石板、火把、丛林色调统一到所有页面。
2. 奖励感前置：碎片、解锁、通关状态要在首屏明确可感知。
3. 交互反馈明确：可拖拽区域、已选择状态、成功/失败反馈一眼可读。
4. 低认知负担：信息分层，主操作按钮始终突出，辅助操作弱化。
5. 移动端优先：保证 390px 宽度下布局完整，不产生横向滚动。

## 3. 视觉系统
- 字体方向：圆润儿童向标题字 + 清晰正文（避免默认系统感）。
- 颜色方向：沙石米色背景 + 丛林绿主按钮 + 琥珀橙奖励色 + 深石棕正文。
- 背景：使用多层渐变 + 轻量纹理，不用纯色平面。
- 卡片：统一“石板卡”风格（边框、阴影、圆角、按下反馈）。
- 动效：
  - 页面进入上浮淡入
  - 地图关卡点位分批出现
  - 可孵蛋按钮轻微脉冲

## 4. 页面改造范围

### 4.1 全局 Layout
- 顶部导航改为“遗迹工具栏”样式，增加品牌标题和副标题。
- 主容器增加统一宽度、留白和入场动画。

### 4.2 地图页 `/`
- 首屏拆为：
  - 左侧（或移动端上方）“任务总览卡”：通关数、碎片、孵蛋进度。
  - 右侧“当前行动卡”：继续挑战 / 前往孵蛋 / 查看图鉴。
- 关卡地图改为“门牌节点”视觉：
  - 锁定 / 可挑战 / 已通关三态颜色与图标明确区分。
  - 节点加入悬停/按压动效。

### 4.3 关卡页 `/level/[id]`
- 信息头部：关卡编号、标题、故事、奖励说明、提示消耗说明。
- 交互区域与侧栏分离：
  - 主区：题面和拖拽/点选区域。
  - 侧栏：提示面板 + 提交栏（移动端回落到下方）。
- 反馈条样式化：成功（绿色）/失败（橙色）视觉区分。

### 4.4 孵蛋页 `/hatch`
- 增加“孵化台”视觉模块（大号蛋形态 + 当前碎片）
- 孵蛋结果单独高亮（解锁动画占位）

### 4.5 图鉴页 `/dex`
- 卡片分为“已解锁/未解锁”两套样式，未解锁保留轮廓占位。
- 已解锁卡显示名称、状态标识。

### 4.6 家长页 `/parent`
- 信息卡片化：解锁关卡、通关数、碎片数。
- 重置按钮弱危险色（避免误触），但仍可见。

## 5. 组件改造
1. `PrimaryButton`
   - 增加尺寸/视觉层次，主按钮突出，ghost 次级。
2. `Card`
   - 强化“可选择 / 已选择 / 可拖拽”状态。
3. `HintPanel`
   - 提示次数可视化（进度点），内容层级清晰。
4. `SubmitBar`
   - 固定主次动作顺序：提交为主，重置为次。

## 6. 验收标准
1. 不改变任何业务流程与关卡判定逻辑。
2. 五个页面统一视觉语言。
3. 地图页、关卡页首屏可读性明显提升。
4. 移动端（390px）和桌面端（>=1280px）均无布局破坏。
5. `npm run build` 通过。
